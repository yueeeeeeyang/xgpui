Rust UI Library

xgpui 使用文档

xgpui 是基于 gpui 的 Rust UI 库，当前已提供 Button 按钮组件、 完整的单行 TextInput 组件和单选 Select 下拉框组件。组件支持内部状态、 外部同步、常见状态样式和明暗皮肤；Button 支持变体、色调、加载和图标， TextInput 额外支持密码、数字、IME、选区、 剪贴板和光标闪烁，Select 额外支持本地搜索、锚定下拉面板和键盘导航。

快速开始

当前项目是单 crate UI 库。使用组件时推荐通过 xgpui::prelude 引入常用类型，并在应用启动时调用 xgpui::install 安装库级默认能力，包括主题状态、Lucide 图标字体和默认键盘绑定。 

// 引入 xgpui 常用组件和配置类型。
use xgpui::prelude::*;

// 在 Application::run 初始化阶段安装 xgpui 默认能力。
Application::new().run(|cx: &mut App| {
    xgpui::install(cx);
    // 在这里继续创建窗口和根视图。
});

必须调用 xgpui::install(cx)，否则退格、删除、方向键、全选、复制、 剪切和粘贴等默认动作不会由 TextInput 接收，Enter、Space、Escape、 方向键、Home 和 End 等默认动作也不会由 Select 接收。该函数是幂等的， 重复调用不会重复注册默认快捷键。

明暗皮肤

xgpui 内置 ThemeMode::Light 和 ThemeMode::Dark 两种皮肤。 皮肤状态存放在 gpui App 全局状态中，所有组件会在渲染时读取同一套主题 token。 

// 安装 xgpui，默认使用 ThemeMode::Light。
xgpui::install(cx);

// 切换到暗色皮肤，并刷新所有窗口。
xgpui::set_theme_mode(cx, ThemeMode::Dark);

// 读取当前皮肤，常用于示例壳层或业务页面同步背景色。
let current = xgpui::theme_mode(cx);

xgpui::install(cx) 不会覆盖已经通过 set_theme_mode 设置过的皮肤。 如果应用希望默认启动为暗色，可以先调用 set_theme_mode(cx, ThemeMode::Dark)， 再调用 install(cx)。

运行示例

项目内置了 examples/button.rs、examples/text_input.rs 和 examples/select.rs， 可直接运行查看组件效果。 

cargo run --example button
cargo run --example text_input
cargo run --example select

button 示例覆盖变体、危险色调、尺寸、禁用、加载、块级按钮、图标和明暗皮肤； text_input 示例覆盖基础输入、密码输入、数字输入、禁用、只读、错误态、helper text、clearable、 前后缀插槽和 set_value 外部同步；select 示例覆盖基础选择、 搜索、禁用、错误态、clearable、空结果、外部同步和明暗皮肤。

Button 基础用法

Button 是 gpui entity 组件，通常在父视图构造函数中通过 cx.new 创建。按钮支持主按钮、次级按钮、描边按钮、幽灵按钮和链接按钮， 并通过 ButtonTone::Danger 表示危险操作。 

/// 示例父视图，持有一个 Button 实体。
struct Toolbar {
    save: Entity<Button>,
}

impl Toolbar {
    /// 创建一个带前置图标的主按钮。
    fn new(cx: &mut Context<Self>) -> Self {
        let save = cx.new(|cx| {
            Button::new(
                cx,
                ButtonProps::default()
                    .label("保存")
                    .leading_icon(LucideIcon::Save)
                    .on_click(|| {
                        // 在这里执行保存动作；loading / disabled 时不会触发。
                    }),
            )
        });

        Self { save }
    }
}

Button 的图标统一使用内置 Lucide 字体。使用图标前仍需调用 xgpui::install(cx)，否则图标字体不会被安装到 gpui 文本系统。

ButtonProps

ButtonProps::default() 提供默认主按钮配置，可通过链式方法覆盖需要的属性。

属性 类型 说明
label SharedString 按钮文案。纯图标按钮会隐藏文案，但会用它作为 tooltip fallback。
variant ButtonVariant Primary、Secondary、Outline、Ghost、Link。
tone ButtonTone Default 或 Danger，用于表达默认操作或危险操作。
size ButtonSize Small、Medium、Large，影响高度、字号、内边距和图标尺寸。
disabled bool 禁用后不可聚焦、不可点击，也不会触发 on_click。
loading bool 展示圆圈 loading 动画并阻止鼠标和键盘触发。
block bool 按钮宽度占满父容器。
icon_only bool 渲染正方形纯图标按钮，建议同时提供图标和 tooltip。
leading_icon / trailing_icon Option<LucideIcon> 前置和后置 Lucide 图标。加载状态会临时用 loading 圆圈动画替代前置图标。
tooltip Option<SharedString> hover 提示文案。纯图标按钮未设置时会回退使用 label。
on_click Option<Box<dyn FnMut()>> 有效点击回调，只在非禁用且非加载时触发。

Button 行为

交互

  • 鼠标点击会先聚焦按钮，再触发 on_click。
  • 按钮聚焦后，点击按钮外部区域会释放焦点并移除焦点边框。
  • 按钮获得焦点后，Enter 和 Space 会触发点击回调。
  • disabled 和 loading 都会阻止点击回调。

视觉

  • 所有颜色来自 foundation::theme::ButtonTheme，支持亮色和暗色皮肤。
  • icon_only 使用正方形尺寸，并隐藏 label 文本。
  • loading 图标使用 gpui 动画系统驱动圆环弧线持续转动，不改变按钮布局尺寸。

基础用法

TextInput 是 gpui entity 组件，通常在父视图构造函数中通过 cx.new 创建，再在 Render 中作为子元素渲染。 

/// 示例父视图，持有一个 TextInput 实体。
struct FormView {
    username: Entity<TextInput>,
}

impl FormView {
    /// 创建输入框并配置占位文本、清除按钮、最大长度和变化回调。
    fn new(cx: &mut Context<Self>) -> Self {
        let username = cx.new(|cx| {
            TextInput::new(
                cx,
                TextInputProps::default()
                    .placeholder("请输入用户名")
                    .clearable(true)
                    .max_length(Some(20))
                    .helper_text(Some(SharedString::from("最多输入 20 个字素簇")))
                    .on_change(|value| {
                        // 在这里同步表单状态或触发上层业务逻辑。
                        println!("当前输入值：{value}");
                    }),
            )
        });

        Self { username }
    }
}

impl Render for FormView {
    /// 渲染父视图内容。
    fn render(&mut self, _window: &mut Window, _cx: &mut Context<Self>) -> impl IntoElement {
        div().child(self.username.clone())
    }
}

输入类型

TextInputType 用于配置输入框内容语义。当前支持 Text、 Password 和 Number；组件值始终是 SharedString， 不会因为类型不同而返回掩码文本或数值类型。 

let password = cx.new(|cx| {
    TextInput::new(
        cx,
        TextInputProps::default()
            .input_type(TextInputType::Password)
            .placeholder("请输入密码")
            .clearable(true),
    )
});

let amount = cx.new(|cx| {
    TextInput::new(
        cx,
        TextInputProps::default()
            .input_type(TextInputType::Number)
            .placeholder("请输入金额"),
    )
});

Password

  • 默认按 Unicode 字素簇显示为掩码字符。
  • 非禁用状态下始终显示眼睛按钮，可切换真实内容展示。
  • value、on_change、复制和剪切都使用真实文本。

Number

  • 允许空值、负号、小数点以及 1.、-.5 等输入中间态。
  • 拒绝字母、多小数点和错误位置的负号。
  • 不做 min、max、step、格式化或数值解析。

TextInputProps

TextInputProps::default() 提供默认配置，可通过链式方法覆盖需要的属性。

属性 类型 说明
value SharedString 初始输入值。创建后需要外部同步时使用 set_value。
placeholder SharedString 空值时显示的占位文本。
disabled bool 禁用后不能聚焦、选择或编辑。
readonly bool 只读后可以聚焦、选择和复制，但不能修改、粘贴、剪切或清除。
clearable bool 有值且非禁用、非只读时显示清除按钮。
required bool 显示必填语义标记。当前版本不执行内置校验。
max_length Option<usize> 最大长度按 Unicode 字素簇计算，粘贴和 IME 输入都会遵守该限制。
size TextInputSize Small、Medium、Large，影响高度、字号、行高和内边距。
variant TextInputVariant Outlined、Filled、Ghost，影响背景和边框展示。
status TextInputStatus Default、Error、Warning、Success，只负责语义颜色。
input_type TextInputType Text、Password、Number，控制密码掩码和数字输入限制。
helper_text Option<SharedString> 输入框下方辅助文本，颜色会跟随语义状态。
prefix Option<TextInputSlot> 前缀插槽，每次渲染生成新的 gpui 元素。
suffix Option<TextInputSlot> 后缀插槽，每次渲染生成新的 gpui 元素。
on_change FnMut(SharedString) 内容确实变化时触发，光标移动、选区变化和焦点变化不会触发。
on_focus / on_blur FnMut() 聚焦和失焦回调。
on_enter FnMut(SharedString) 按下回车时传出当前完整文本，适合搜索和简单提交。
on_key_down FnMut(Keystroke) 透出 gpui 按键事件，方便上层记录或响应特殊按键。

前后缀插槽

TextInputSlot 保存的是可重复调用的闭包，每次渲染都会生成新的 AnyElement。这样可以避免持有一次性 gpui element 引发的生命周期和重复渲染问题。 

let url_input = cx.new(|cx| {
    TextInput::new(
        cx,
        TextInputProps::default()
            .placeholder("example.com")
            .prefix(Some(TextInputSlot::new(|| {
                // 前缀可以是任意 gpui 元素。
                div().child("https://").into_any_element()
            })))
            .suffix(Some(TextInputSlot::text(".com")))
            .clearable(true),
    )
});

公开方法

方法 用途
TextInput::new(cx, props) 创建组件状态。
value() 读取当前文本值。
set_value(value, cx) 从外部同步文本值，不触发 on_change，避免同步回环。
clear(cx) 清空文本，并在内容变化时触发 on_change。
focus_handle() 返回 gpui 焦点句柄，便于父组件主动聚焦或协调焦点。
select_all(cx) 选中全部文本。
move_to_end(cx) 把光标移动到文本末尾。 
self.username.update(cx, |input, cx| {
    // 外部写入值后移动到末尾，适合受控表单回填场景。
    input.set_value("由父组件写入", cx);
    input.move_to_end(cx);
});

键盘与输入行为

文本编辑

  • 支持单行输入，粘贴多行文本时会把换行规范化为空格。
  • 支持 Backspace、Delete、方向键、Home、End、全选、复制、剪切和粘贴。
  • 支持中文输入法 marked text 和候选框范围定位。
  • 密码类型只改变显示文本，不改变真实值和剪贴板文本。
  • 数字类型会拒绝不符合数字形态的键盘输入、粘贴和 IME 提交。

选择与滚动

  • 支持鼠标定位、拖选和 Shift 扩展选择。
  • 拖选到输入框边缘时会自动横向滚动并继续扩展选区。
  • 光标静止后会闪烁，移动或输入时会恢复常亮。

Unicode 处理

  • 光标移动按 Unicode 字素簇边界处理。
  • max_length 按用户可感知字素簇计数。
  • 内部包含 UTF-8 和 UTF-16 区间转换，用于平台输入协议。

状态限制

  • disabled 禁止聚焦、选择和编辑。
  • readonly 允许聚焦、选择和复制，但禁止修改。
  • status 只展示语义颜色，不内置表单校验器。

Select 基础用法

Select 是 gpui entity 组件，通常在父视图中通过 cx.new 创建。第一版只支持单选、本地搜索和文本 option，不支持多选、远程加载、分组或自定义 option 渲染。 

/// 示例父视图，持有一个 Select 实体。
struct FormView {
    city: Entity<Select>,
}

impl FormView {
    /// 创建城市选择框并配置选项、搜索、清除和变化回调。
    fn new(cx: &mut Context<Self>) -> Self {
        let city = cx.new(|cx| {
            Select::new(
                cx,
                SelectProps::default()
                    .placeholder("请选择城市")
                    .options(vec![
                        SelectOption::new("beijing", "北京"),
                        SelectOption::new("shanghai", "上海"),
                        SelectOption::new("disabled", "不可选城市").disabled(true),
                    ])
                    .searchable(true)
                    .clearable(true)
                    .helper_text(Some(SharedString::from("支持键盘导航和本地搜索")))
                    .on_change(|value| {
                        // 在这里同步表单状态或触发上层业务逻辑。
                        println!("当前选择值：{value:?}");
                    }),
            )
        });

        Self { city }
    }
}

impl Render for FormView {
    /// 渲染父视图内容。
    fn render(&mut self, _window: &mut Window, _cx: &mut Context<Self>) -> impl IntoElement {
        div().child(self.city.clone())
    }
}

SelectOption::value 建议在同组选项中保持唯一。如果出现重复值，组件会按第一个 匹配项展示和选择，避免运行时额外报错破坏现有数据流。

SelectProps

SelectProps::default() 提供默认配置，可通过链式方法覆盖需要的属性。

属性 类型 说明
value Option<SharedString> 初始选中值。创建后需要外部同步时使用 set_value。
placeholder SharedString 未选择时显示的占位文本。
options Vec<SelectOption> 下拉选项列表。SelectOption 包含 value、label 和 disabled。
disabled bool 禁用后不能聚焦、打开、选择或清除。
clearable bool 有值且非禁用时显示清除按钮。
required bool 显示必填语义标记。当前版本不执行内置校验。
searchable bool 打开面板后直接在原选择框内输入内容，按 label 做大小写不敏感子串匹配。
search_placeholder SharedString 搜索模式下原选择框为空时显示的占位文本。
size SelectSize Small、Medium、Large，影响触发器和选项尺寸。
variant SelectVariant Outlined、Filled、Ghost，影响触发器背景和边框。
status SelectStatus Default、Error、Warning、Success，只负责语义颜色。
helper_text Option<SharedString> 选择框下方辅助文本，颜色会跟随语义状态。
max_popup_height Pixels 下拉面板最大高度，超过后内部滚动。
empty_text SharedString 选项为空或搜索结果为空时展示的文本。
on_change FnMut(Option<SharedString>) 用户选择或清除导致值变化时触发，外部 set_value 不触发。
on_open_change FnMut(bool) 下拉面板打开状态变化时触发。
on_search_change FnMut(SharedString) 搜索词变化时触发。

Select 公开方法

方法 用途
Select::new(cx, props) 创建组件状态。
value() 读取当前选中值。
set_value(value, cx) 从外部同步选中值，不触发 on_change，避免同步回环。
clear(cx) 清空当前选择，并在值变化时触发 on_change。
open(cx) / close(cx) / toggle(cx) 从外部控制下拉面板打开状态。

self.city.update(cx, |select, cx| {
    // 外部写入值不会触发 on_change，适合受控表单回填场景。
    select.set_value(Some(SharedString::from("shanghai")), cx);
});

Select 键盘行为

打开与选择

  • Enter / Space：关闭时打开，打开时选择当前高亮项。
  • Escape：关闭下拉面板。
  • Tab：保留系统焦点切换行为，同时关闭下拉面板。

导航

  • ArrowDown / ArrowUp：打开面板并移动高亮项。
  • Home / End：移动到第一个或最后一个可选项。
  • 键盘高亮会跳过禁用选项。

搜索

  • searchable = true 时，打开面板会清空搜索词。
  • 输入普通字符会直接显示在原选择框内，并按 label 进行大小写不敏感本地过滤。
  • 搜索编辑由 Select 内部实现，支持光标闪烁、移动、选区、拖拽选中、复制和粘贴。
  • Backspace 会删除搜索词末尾的一个 Unicode 字素簇。

面板

  • 下拉面板通过 gpui::anchored() 锚定到触发器下方。
  • 超过 max_popup_height 后内部滚动。
  • 点击组件外部会关闭面板；当前版本不主动做上下翻转。

实现约束

  • Button 第一版支持常用单按钮能力，不包含按钮组、异步状态管理或任意元素插槽。
  • 当前版本只实现单行 TextInput，包含普通文本、密码和数字类型，不包含 email、search 或 textarea。
  • 当前版本只实现单选 Select，不包含多选、远程加载、分组、虚拟列表或自定义 option 渲染。
  • 项目优先使用稳定 Rust，不引入 nightly 特性。
  • 新功能应按模块、类型和职责拆分，避免把组件逻辑堆在单个文件或单个方法中。
  • 公共 API、组件行为、状态流转和边界条件应通过中文注释说明设计意图。
  • 所有新增或扩展组件都需要同步实现明暗皮肤，并通过 foundation::theme 解析颜色 token。

如果需要新增按钮组、按钮任意元素插槽、表单校验、多行文本、多选 Select 或远程搜索， 请先补充交互需求和 API 预期， 再扩展组件层次和状态模型。
